# Local Quiz App

業務知識や資格試験対策のための、Windows向けオフラインクイズアプリケーション。
ネットワーク接続を一切必要とせず、ローカル環境で完結する軽量な学習ツールです。

## 特徴

- **完全オフライン** — 外部ネットワークへの通信を一切行いません
- **軽量・高速** — Tauri v2 (Rust) ベースで起動・動作が高速
- **アプリ内で作問・編集** — GUIエディタでクイズパックを作成・編集できます。JSONインポートにも対応
- **4種類の出題形式** — 択一選択、複数選択、○×、テキスト入力に対応
- **学習分析** — 正答率の推移や弱点問題を可視化し、効率的な復習をサポート
- **エクスポート** — 作成したクイズパックをJSONファイルとして書き出し・共有可能
- **ダークモード対応** — ライト/ダークテーマを切り替え可能

## 用途例

- 業務に必要なドメイン知識（法規、製品知識、社内ルール等）の反復学習
- 資格試験の対策用ドリル
- チームや部門向けの教育ツール（管理者がクイズパックを作成・エクスポートして配布）

## 動作環境

- Windows 10 以降
- WebView2 ランタイム（Windows 10/11 には標準搭載）

## インストール

1. [Releases](https://github.com/llrhino/local-quiz-app/releases) ページから最新のインストーラー（`.msi` または `.exe`）をダウンロード
2. インストーラーを実行し、画面の指示に従ってインストール
3. スタートメニューまたはデスクトップショートカットからアプリを起動

## 操作方法

### クイズパックの作成・編集

1. ホーム画面で「新規作成」ボタンをクリック
2. パック名・説明を入力し、問題を追加していく
3. 問題タイプ（択一選択 / 複数選択 / ○× / テキスト入力）を選んで問題文・選択肢・正答・解説を入力
4. 問題の並び替えや削除も可能
5. 「保存」ボタンで保存（編集中のデータは自動バックアップされるため、不意のクラッシュでも復元可能）

既存のクイズパックは一覧から選択して編集できます。

### クイズパックのインポート・エクスポート

- **インポート:** ホーム画面の「インポート」ボタンからJSONファイルを選択。同じパックIDのものが既にある場合は上書き更新も可能
- **エクスポート:** パックの詳細画面からJSONファイルとして書き出し。他のユーザーへの配布に利用できます
- **サンプルパック:** 初回利用時にサンプルクイズパックを読み込めます

### クイズの実行

1. ホーム画面でクイズパックを選択して「開始」
2. 出題された問題に回答（選択肢クリック / ○×クリック / テキスト入力）
3. 回答後、正誤と解説が表示される
4. 全問完了後、正答率と回答一覧がサマリー画面に表示される
5. セッション途中で中断した場合も、それまでの回答は履歴に保存されます

### 学習履歴・弱点分析

- パックごとの全体統計（総回答数、正解数、正答率）を確認可能
- セッション単位の学習記録一覧
- **弱点問題の自動抽出** — 2回以上回答した問題のうち正答率が100%未満のものを弱点として表示。問題文、正答率、あなたの回答と正解を一覧で確認できます

### アプリ設定

| 設定項目         | 選択肢            | デフォルト |
| ---------------- | ----------------- | ---------- |
| 出題順           | 定義順 / ランダム | 定義順     |
| 選択肢シャッフル | オン / オフ       | オフ       |
| テーマ           | ライト / ダーク   | ライト     |

## クイズパックのJSON形式

アプリ内のGUIエディタで作成するほか、JSONファイルを直接作成してインポートすることもできます。

```json
{
  "pack": {
    "id": "example-pack-001",
    "name": "サンプルクイズ",
    "description": "クイズパックの説明文"
  },
  "questions": [
    {
      "id": "q-001",
      "type": "multiple_choice",
      "question": "次の文章を読んで答えてください。\n\n日本の首都はどこですか？",
      "choices": [
        { "text": "大阪" },
        { "text": "東京" },
        { "text": "京都" },
        { "text": "名古屋" }
      ],
      "answer": 1,
      "explanation": "日本の首都は東京です。"
    },
    {
      "id": "q-002",
      "type": "true_false",
      "question": "水の化学式はH2Oである。",
      "answer": true,
      "explanation": "水の化学式はH2Oです。"
    },
    {
      "id": "q-003",
      "type": "text_input",
      "question": "日本で一番高い山の名前を入力してください。",
      "answer": "富士山"
    },
    {
      "id": "q-004",
      "type": "multi_select",
      "question": "次のうち、プログラミング言語であるものをすべて選んでください。",
      "choices": [
        { "text": "Python" },
        { "text": "HTML" },
        { "text": "Rust" },
        { "text": "SQL" }
      ],
      "answer": [0, 2],
      "explanation": "PythonとRustはプログラミング言語です。HTMLはマークアップ言語、SQLはクエリ言語です。"
    }
  ]
}
```

> **ヒント:** 問題文や解説に改行を入れたい場合は、JSON文字列内で `\n` を使用してください（上記 `q-001` の例を参照）。

### `questions.id` のルール

- `questions.id` は各設問を識別するためのIDです。`questions` 配列の中で重複しない必要があります。
- 形式は任意です。`q-001` のような連番でも、`network-basic-001` のような意味のある文字列でも構いません。
- 同じパックを後で更新インポートする場合は、同じ設問に同じ `id` を使い続けてください。更新時の突合や学習履歴の扱いに使われます。
- 設問を別物として扱いたい場合だけ `id` を変更してください。
- GUIエディタで保存した場合は、アプリ側で `q1`, `q2`, ... のように再採番されることがあります。手動でJSONを管理する場合は、読みやすく安定したIDを付けるのがおすすめです。

### 問題タイプ

| タイプ            | 説明                         | `answer` の形式                                |
| ----------------- | ---------------------------- | ---------------------------------------------- |
| `multiple_choice` | 択一選択（2〜4択）           | 正解の選択肢インデックス（例: `1`）            |
| `multi_select`    | 複数選択（2〜4択、複数正解） | 正解の選択肢インデックスの配列（例: `[0, 2]`） |
| `true_false`      | ○×式                         | `true` または `false`                          |
| `text_input`      | テキスト入力式               | 正解テキスト（文字列、大文字小文字を区別）     |

> **注意:** 選択肢のインデックスは **0始まり** です。1番目の選択肢が `0`、2番目が `1`、3番目が `2`、4番目が `3` になります。

## セキュリティ

- **ネットワーク通信なし** — アプリは外部への通信を一切行いません。Tauri の CSP（Content Security Policy）設定により、外部リソースへのアクセスをブロックしています
- **データはローカルに保存** — すべてのデータ（クイズパック、学習履歴、設定）はローカルの SQLite データベースに保存されます。外部サーバーにデータが送信されることはありません
- **安全なファイルインポート** — インポート時はJSONパースとバリデーションのみを行い、スクリプト実行等のリスクを排除しています
