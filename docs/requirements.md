# ローカルクイズアプリ 要件定義書

**バージョン:** 1.0
**作成日:** 2026-03-10
**ステータス:** 初版

---

## 1. プロジェクト概要

### 1.1 プロジェクト名

Local Quiz App

### 1.2 目的

業務やドメイン知識の継続的な学習を支援するための、Windows向けローカルデスクトップクイズアプリケーションを開発する。外部ネットワークへの接続を一切必要とせず、ユーザーのローカル環境で完結する軽量な学習ツールを提供する。

### 1.3 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | React + TypeScript |
| ビルドツール | Vite |
| デスクトップフレームワーク | Tauri v2 |
| データ永続化 | SQLite |
| 対象OS | Windows（10以降） |

### 1.4 用語定義

| 用語 | 定義 |
|---|---|
| クイズパック | 問題群を一つのまとまりとして管理する単位。単一のJSONファイルとして配布・インポートされる |
| 学習セッション | ユーザーがクイズパックを選択し、出題から結果確認までを行う一連の学習活動 |

---

## 2. コンセプトとビジョン

### 2.1 コンセプト

**「静かなローカル知識トレーニンググラウンド」**

日々の業務の合間や学習時間に、静かに自分のペースで知識の定着度を確認し、弱点を把握・克服するための個人トレーニング環境。

### 2.2 設計原則

1. **オフラインファースト** — ネットワーク接続を前提としない
2. **データオーナーシップ** — ユーザーが自身のデータを完全にコントロールできる
3. **低い導入障壁** — インストールして即使い始められる
4. **シンプルさ** — 問題→回答→結果→解説のサイクルに集中できるミニマルなUI

### 2.3 体験の方針

- **Fast** — すぐに問題に取り組める
- **Focused** — 不要なUI要素を排除
- **Private** — 完全オフライン、ローカルデータ
- **Constructive** — 間違いが学びにつながる
- **Accumulating** — 知識の蓄積を実感できる

トーンは落ち着いた・実用的な雰囲気。軽度のゲーミフィケーションはあるが、過度な演出はしない。

## 3. ターゲットユーザー

### 3.1 プライマリユーザー

- 業務に必要なドメイン知識（法規、製品知識、社内ルール等）を継続的に学習・復習したい社会人
- 資格試験の対策として反復学習を行いたい個人

### 3.2 セカンダリユーザー

- チームや部門向けにクイズパックを作成・配布する教育担当者

### 3.3 ユーザー特性

- ITリテラシー: 中程度（日常的にPCを使用するが、開発経験は不要）
- 利用シーン: 業務の隙間時間、始業前・終業後の短時間学習

## 4. 機能要件（MVP）

### 4.1 クイズパック管理

#### FR-PACK-001: クイズパックのインポート

| 項目 | 内容 |
|---|---|
| 概要 | ユーザーがローカルのJSONファイルを選択し、クイズパックとしてアプリにインポートする |
| トリガー | ユーザーが「インポート」ボタンを押下 |
| 入力 | 単一のJSONファイル（`.json`拡張子） |
| 処理 | 1. ファイル選択ダイアログを表示 2. JSONファイルを読み込み 3. バリデーション実行（FR-PACK-002） 4. 成功時、アプリ内データストアに保存 5. 結果を通知 |
| 出力 | インポート成功/失敗メッセージ |

#### FR-PACK-002: インポートバリデーション

インポート時に以下のバリデーションを実行する。1件でもエラーがあればインポート全体を拒否する。

| バリデーション項目 | ルール | エラー時の表示 |
|---|---|---|
| JSON構文 | 有効なJSON形式であること | 構文エラー箇所を表示 |
| 必須フィールド | `pack`（`id`, `name`）, `questions` が存在すること | 不足フィールド名を表示 |
| パックID一意性 | 既にインポート済みのパックIDと重複しないこと | 重複パックIDを表示 |
| 問題タイプ | 各問題の `type` が `multiple_choice`, `true_false`, `text_input` のいずれかであること | 不正な問題IDとtype値を表示 |
| 選択肢整合性 | `multiple_choice` の場合、`choices` が2〜4個であること | 不正な問題IDを表示 |
| 正答整合性 | `answer` が選択肢のid内に存在すること（multiple_choice）/ `true`or`false`であること（true_false）/ 空でないこと（text_input） | 不正な問題IDとanswerを表示 |
| 問題ID一意性 | パック内で問題IDが重複しないこと | 重複IDを表示 |

エラーメッセージの形式:

```
Question ID: net-002
Field: answer
Error: invalid value
```

#### FR-PACK-003: クイズパック一覧表示

| 項目 | 内容 |
|---|---|
| 概要 | インポート済みのクイズパックを一覧表示する |
| 表示項目 | パック名、説明（任意）、問題数、インポート日時、最終学習日時（履歴がある場合） |
| ソート | インポート日時の降順（デフォルト） |

#### FR-PACK-004: クイズパックの削除

| 項目 | 内容 |
|---|---|
| 概要 | インポート済みのクイズパックを削除する |
| 処理 | 確認ダイアログ表示後、パックデータおよび関連する学習履歴を物理削除する |

### 4.2 クイズ実行

#### FR-QUIZ-001: クイズセッションの開始

| 項目 | 内容 |
|---|---|
| 概要 | ユーザーがクイズパックを選択し、学習セッションを開始する |
| 処理 | パック内の全問題を出題対象とする。出題順はパック内定義順（デフォルト）またはランダム（設定による） |

#### FR-QUIZ-002: 問題の出題と回答

問題タイプごとの出題形式:

| 問題タイプ | 出題UI | 回答方法 |
|---|---|---|
| `multiple_choice` | 問題文 + 最大4つの選択肢ボタン | 選択肢をクリック |
| `true_false` | 問題文 + ○/×ボタン | ○または×をクリック |
| `text_input` | 問題文 + テキスト入力フィールド | テキストを入力して送信 |

共通動作:
- 回答は1問につき1回のみ（再回答不可）
- 回答後、即座に正誤判定を実行する

#### FR-QUIZ-003: 正誤判定ルール

| 問題タイプ | 判定ルール |
|---|---|
| `multiple_choice` | ユーザーが選択した選択肢のIDが `answer` と一致すること |
| `true_false` | ユーザーの回答が `answer` と一致すること |
| `text_input` | `userAnswer.trim() === correctAnswer.trim()`（**大文字小文字は区別する。追加の正規化は行わない**） |

#### FR-QUIZ-004: 結果・解説表示

| 項目 | 内容 |
|---|---|
| 結果表示 | 正解/不正解の判定、正答、ユーザーの回答 |
| 解説表示 | 問題に `explanation` が設定されている場合に表示。未設定時は「解説はありません」 |
| 動作 | 「次の問題へ」で次に進む |

#### FR-QUIZ-005: セッション完了

| 項目 | 内容 |
|---|---|
| 表示項目 | 総問題数、正解数、正答率、回答一覧（問題番号・正誤） |
| 動作 | 「パック一覧に戻る」「もう一度挑戦する」を選択可能 |

#### FR-QUIZ-006: セッションの中断

| 項目 | 内容 |
|---|---|
| 概要 | ユーザーはセッション途中で中断できる |
| 処理 | 中断時点までの回答は学習履歴として保存する。途中再開機能はMVPでは提供しない |

---

### 4.3 学習履歴

#### FR-HIST-001: 学習履歴の記録

| 項目 | 内容 |
|---|---|
| 記録タイミング | **各問題の回答確定時**（1問ごとに即座に保存） |
| 記録項目 | パックID、問題ID、正誤、ユーザーの回答値、タイムスタンプ |

#### FR-HIST-002: 学習履歴の閲覧

| 項目 | 内容 |
|---|---|
| 概要 | クイズパックごとの学習履歴を閲覧できる |
| 表示項目 | セッション日時、正答率、問題ごとの正誤推移 |

#### FR-HIST-003: 弱点把握

| 項目 | 内容 |
|---|---|
| 概要 | 誤答率の高い問題を抽出し、弱点として可視化する |
| 表示項目 | 問題文（または問題ID）、回答回数、正答率、直近のユーザー回答値 |
| ソート | 正答率の昇順 |
| 条件 | 最低2回以上回答した問題を対象 |

|---|---|---|

## 5. 非機能要件

### 5.1 性能

| 項目 | 要件 |
|---|---|
| アプリ起動時間 | 3秒以内 |
| 画面遷移 | 200ms以内 |
| インポート処理 | 1000問規模で3秒以内 |

### 5.2 セキュリティ

| 項目 | 要件 |
|---|---|
| ネットワーク通信 | 一切行わない。Tauriの設定で外部通信を無効化する |
| データ保存 | ローカルファイルシステムに保存（暗号化はMVP外） |
| インポートファイル | JSONパースのみ。スクリプト実行等のリスクを排除する |

### 5.3 ユーザビリティ

| 項目 | 要件 |
|---|---|
| キーボード操作 | クイズ回答がキーボードのみで完結できること |
| フィードバック | 処理中はローディング表示を行うこと |
| エラーメッセージ | ユーザーが次のアクションを判断できる具体的なメッセージ |

## 6. 画面一覧と画面遷移

### 6.1 画面一覧

| 画面ID | 画面名 | 概要 |
|---|---|---|
| SCR-HOME | ホーム画面 | インポート済みクイズパック一覧。インポート・削除等のアクション |
| SCR-QUIZ | クイズ画面 | 問題出題・回答入力。進捗表示（例: 3/20） |
| SCR-RESULT | 結果画面 | 回答直後の正誤・解説表示 |
| SCR-SUMMARY | サマリー画面 | セッション全体の結果一覧と正答率 |
| SCR-HISTORY | 学習履歴画面 | パックごとの学習履歴・弱点表示 |
| SCR-SETTINGS | 設定画面 | アプリ設定の変更 |

### 6.2 画面遷移

```
SCR-HOME
  ├── [インポート] → ファイル選択ダイアログ → 結果通知 → SCR-HOME
  ├── [パック選択・開始] → SCR-QUIZ
  │     ├── [回答] → SCR-RESULT
  │     │     └── [次の問題] → SCR-QUIZ
  │     ├── [全問完了] → SCR-SUMMARY
  │     │     ├── [パック一覧に戻る] → SCR-HOME
  │     │     └── [もう一度挑戦] → SCR-QUIZ
  │     └── [中断] → SCR-HOME
  ├── [履歴] → SCR-HISTORY → [戻る] → SCR-HOME
  └── [設定] → SCR-SETTINGS → [戻る] → SCR-HOME
```

## 7. クイズパックJSONスキーマ定義

### 7.1 トップレベル構造

```json
{
  "pack": { ... },
  "questions": [ ... ]
}
```

### 7.2 `pack` オブジェクト

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | Yes | パックの一意識別子 |
| `name` | string | Yes | パック表示名 |
| `description` | string | No | パックの説明文 |

### 7.3 `questions` 配列

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | Yes | 問題の一意識別子（パック内で一意） |
| `type` | string | Yes | `"multiple_choice"`, `"true_false"`, `"text_input"` |
| `question` | string | Yes | 問題文 |
| `choices` | array of Choice | Conditional | `type` が `multiple_choice` の場合に必須（2〜4要素） |
| `answer` | string / boolean | Yes | 正答（型は `type` により異なる） |
| `explanation` | string | No | 解説文 |

**Choice:**

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | Yes | 選択肢ID |
| `text` | string | Yes | 選択肢テキスト |

**`answer` の型ルール:**

| 問題タイプ | `answer` の型 | 制約 |
|---|---|---|
| `multiple_choice` | string | `choices` 内のいずれかの `id` と一致 |
| `true_false` | boolean | `true` または `false` |
| `text_input` | string | 空文字列でないこと |

### 7.4 JSONサンプル

```json
{
  "pack": {
    "id": "security-basics-001",
    "name": "情報セキュリティ基礎",
    "description": "情報セキュリティに関する基礎知識を問うクイズパック"
  },
  "questions": [
    {
      "id": "q-001",
      "type": "multiple_choice",
      "question": "AESの鍵長として正しくないものはどれか？",
      "choices": [
        { "id": "a", "text": "128ビット" },
        { "id": "b", "text": "192ビット" },
        { "id": "c", "text": "256ビット" },
        { "id": "d", "text": "512ビット" }
      ],
      "answer": "d",
      "explanation": "AESの鍵長は128, 192, 256ビットの3種類です。512ビットは規格に存在しません。"
    },
    {
      "id": "q-002",
      "type": "true_false",
      "question": "多要素認証は、同じ種類の認証要素を複数組み合わせることである。",
      "answer": false,
      "explanation": "多要素認証は「知識」「所持」「生体」など、異なる種類の認証要素を組み合わせるものです。"
    },
    {
      "id": "q-003",
      "type": "text_input",
      "question": "公開鍵暗号方式の代表的なアルゴリズムで、3人の開発者の頭文字からなるものは何か？（アルファベット3文字）",
      "answer": "RSA",
      "explanation": "Rivest, Shamir, Adlemanの3名によって開発されました。"
    }
  ]
}
```

## 8. データモデル

### 8.1 quiz_packs

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | TEXT | PK | パックID（JSONの `pack.id`） |
| name | TEXT | NOT NULL | パック表示名 |
| description | TEXT | | パック説明文 |
| question_count | INTEGER | NOT NULL | 問題数 |
| imported_at | TEXT | NOT NULL | インポート日時（ISO 8601） |

### 8.2 questions

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | TEXT | PK | `{pack_id}_{question_id}` の複合キー |
| pack_id | TEXT | NOT NULL, FK | クイズパックID |
| question_id | TEXT | NOT NULL | パック内の問題ID |
| question_type | TEXT | NOT NULL | 問題タイプ |
| question_text | TEXT | NOT NULL | 問題文 |
| choices_json | TEXT | | 選択肢のJSON（multiple_choiceの場合） |
| correct_answer | TEXT | NOT NULL | 正答（DB保存時は全て文字列に変換） |
| explanation | TEXT | | 解説文 |
| sort_order | INTEGER | NOT NULL | 出題順 |

### 8.3 learning_history

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | INTEGER | PK, AUTO INCREMENT | レコードID |
| pack_id | TEXT | NOT NULL, FK | クイズパックID |
| question_id | TEXT | NOT NULL | パック内の問題ID |
| is_correct | INTEGER | NOT NULL | 正誤（0=不正解, 1=正解） |
| user_answer | TEXT | NOT NULL | ユーザーの回答値 |
| answered_at | TEXT | NOT NULL | 回答日時（ISO 8601） |

インデックス:
- `idx_history_pack_question` ON (`pack_id`, `question_id`)
- `idx_history_answered_at` ON (`answered_at`)

### 8.4 app_settings

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| key | TEXT | PK | 設定キー |
| value | TEXT | NOT NULL | 設定値 |

初期値:

| key | value | 説明 |
|---|---|---|
| `question_order` | `"sequential"` | 出題順 |
| `theme` | `"light"` | テーマ |

## 9. MVP除外事項

| 項目 | 除外理由 |
|---|---|
| ユーザー認証・アカウント管理 | ローカル完結・シングルユーザー前提 |
| クラウド同期 | オフラインファースト方針 |
| ソーシャル機能 | コンセプトに合致しない |
| 複雑な採点ロジック | trim()による完全一致で十分 |
| アプリ内クイズパック作成・編集 | JSONファイル直接作成で対応 |
| クイズパックのエクスポート | インポートのみで運用可能 |
| 学習スケジュール・リマインダー | 手動学習のみ |
| データの暗号化 | ローカル保存のため優先度低 |
| Windows以外のOS | MVPをWindowsに限定 |
| カテゴリ・マスター機能 | 作問ハードルを下げるためMVPでは導入しない。将来スキーマバージョンを上げて対応 |
| セッションの途中再開 | 中断時は回答済み分のみ保存 |
